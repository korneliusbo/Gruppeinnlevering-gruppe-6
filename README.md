### 3.1 Partnerskapskontrakter
I prosjektledelse trekker man til partnerskapskontrakten som den tradisjonelle måten å tenke på. Partnerskapskontrakter baserer seg på de teknisk materielle bidragene til de enkelte prosjektdeltakere i gruppen, i en slik kontrakt formulerer det hva som forventes av de ulike partene og hva slags forventninger de har til hverandre. (Jacobsen, 2016, s.43). Slike kontrakter handler om tid, kostnad og kvalitet. Altså "jerntranglet".  Derfor vil kontrakten først og fremst bestemme hvor mye  prosjektet får lov å koste, tidsgrensen for prosjektet, og hvilken funksjonalitet eller kvalitet som skal oppnås. 

For Erlings digitale medborgerportal kan dette innebære et definert budsjett innenfor IT-avdelingen som bestemmer hvor mye penger som skal gå til utviklingen. Det kan også innebære hvor mange timer UX- designeren har å bruke, samt hvilke oppgaver/leveranser som skal være ferdig til bestemte tider. Rammer slik som dette vil gi trygghet og kan være nyttige for helhetlig kontroll og fremdrift i prosjektet.  

Partnerskapskontrakter er primært rettet mot de økonomiske og tekniske sidene av et prosjekt, dette er det utfordringen kan oppstå. Det gir blant annet lite føringer på hvordan aktørene skal samarbeide, hvordan fleksibilitet skal bygges og hvordan konflikter skal håndteres. Medborgerportalen kan defineres som en vanskelig satsing, på grunn av ting som, politisk føring, medvirkning på innbyggere og faglige hensyn som skal ivaretas. Jerntriangelet er derfor ikke tilstrekkelig og støtte seg fullt og helt alene på.

Å supplere tradisjonelle kontraktsrammer med relasjonelle kontrakter kan derfor være lurt i prosjektet, fordi det vil legge mer vekt på tillit, dialog og samarbeid. Dette kan være avgjørende for et godt samspill mellom enkelte i prosjektet, og prosjekteieren som har ledelsen over Erling igjen.
